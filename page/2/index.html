RxBinding是什么

把 Android 中各种 UI 控件的事件转换为 RxJava 中的数据流。这样就可以把 UI 控件的事件当做 RxJava 中的数据流来使用了。 比如 View 的 onClick 事件，使用 RxView.clicks(view) 即可获取到一个 Observable 对象，每当用户点击这个 View 的时候，该 Observable 对象就发射一个事件（onNext 被调用）， Observable 的 Observer 订阅者就可以通过 onNext 回调知道用户点击了 View。

RxAndroid

Android开发的必须元素，里面知识提供了两个简单的功能，AndroidSchedulers.mainThread()
AndroidSchedulers.handlerThread(handler)，Android开发最核心的功能。

RxBinding

RxBinding 是把 Android 中各种 UI 控件的事件转换为 RxJava 中的数据流。这样就可以把 UI 控件的事件当做 RxJava 中的数据流来使用了。 比如 View 的 onClick 事件，使用 RxView.clicks(view) 即可获取到一个 Observable 对象，每当用户点击这个 View 的时候，该 Observable 对象就发射一个事件（onNext 被调用）， Observable 的 Observer 订阅者就可以通过 onNext 回调知道用户点击了 View。

具体介绍点这里

RxLifecyle

RxLifecycle 配合 Activity/Fragment 生命周期来管理订阅的。 由于 RxJava Observable 订阅后（调用 subscribe 函数），一般会在后台线程执行一些操作（比如访问网络请求数据），当后台操作返回后，调用 Observer 的 onNext 等函数，然后在 更新 UI 状态。 但是后台线程请求是需要时间的，如果用户点击刷新按钮请求新的微博信息，在刷新还没有完成的时候，用户退出了当前界面返回前面的界面，这个时候刷新的 Observable 如果不取消订阅，则会导致之前的 Activity 无法被 JVM 回收导致内存泄露。 这就是 Android 里面的生命周期管理需要注意的地方，RxLifecycle 就是用来干这事的。比如下面的示例：

RxBus

RxBus并不是一个库，而是一种模式。相信大多数开发者都使用过EventBus，作为事件总线通信库，如果你的项目已经加入RxJava和EventBus，不妨用RxBus代替EventBus，以减少库的依赖。

还是推荐大家使用系统自带的广播吧

RxPermission

是基于RxJava开发的用于帮助在Android 6.0中处理运行时权限检测的框架。在Android 6.0中，系统新增了部分权限的运行时动态获取。而不再是在以前的版本中安装的时候授予权限。

我的2016总结

发表于 2017-01-01 | 分类于 随笔 | 阅读次数

整整一年我也只是在最后一段时间才找到了自己的方向有了自己的目标，才知道自己该做什么。

寒假期间，完成实验室布置的寒假作业，一个很简单的计算器，很好笑的是那个时候连button都用不好，控件什么的更是一塌糊涂，于是用了寒假培训时候的surfaceView把每一个控件都可以说是画了出来，其实现在也知道这也是这些控件的底层原理，一个没有数据结构，只用逻辑思维的简陋计算器就这么写完了，还是有很多的bug，开学之后开始《第一行代码》《Android“字典”》跟着上面一直写，也不知道自己在学什么，想要学好，但是一直没有脱坑。四月初的一个电话把我从迷茫中敲醒，学长让我给院里做一个关于四进四信的app，知道这个任务的时候一脸懵逼。因为感觉自己真的什么也不会，于是开始慢慢构思。在半个与之后，一个很简单，只有页面跳转，listview，scrollview和一写简单控件的app写了出来，自己现在想起来也是笑笑傻傻的自己。自己第一个“丑陋”的app。

之后就是实验室的分组首先是为了参加之后的大学生创业项目，于是乎五个可爱的人组在一起，也就是后来的知行小组，我们也没有什么想法，也就随便想了想，但是被老师给b掉了，当时几个人也是有点沮丧，因为同时大一的两外两个组都通过了。后来也就是暑假时候的我们为了准备华北五省比赛吧，暑假期间选择了回家先待一段时间，后来回学校学习，本来准备一天写一篇github的东西，但是发现好难，一方面自己没有办法全部掌握，另一方面没有办法坚持下来，可能是自己太懒惰了吧。快开学的时候也是开始跟着写比赛的东西，还记得刚刚来实验室，看到学长做出地图觉得好厉害，然后。。。。。。嘻嘻

之后的校赛答辩可以说是很幸运，也可能是秦老师自己的项目吧，我们以最后一名的身份参加省赛，也是暂时性的松了一口气，国庆假期，在实验室和小组一起写没有完成的功能，那些日子真的感觉自己快死了，一天一顿饭。不过好在写了出来。之后的模拟答辩更是惨不忍睹，至今记得秦老师站起来批评的样子，老师也是很希望我们表现更好的，于是有了我们的三次模拟答辩。省赛我们表现的很好，和一等奖只差一点点，不过也是圆了我们可以一起去北京的愿望吧，之间还有一个大一新生的面试，看着他们进来我们好像看到了曾经的自己，也是那么的稚嫩，但是却掩盖不住他们的自信，他们每一个都想成为最好的，也很努力。之后他们也表现的很好，可以说，比我们那届要好得多。希望他们明年会有很好的表现，把android实验室更好的推广出去。

去了北京之后，一方面很开心，另一方面很紧张。也许我们真的是朝着二等奖去的吧，答辩我们做得很好，对了还有我们一起去吃了好多好吃的，几个大一的小伙伴和乔神，文慧姐，阿哲学长的火锅，还有斌哥的夜宵（可以说是大学里的亲哥哥），还有实验室所有人的聚餐。最后的一等奖可以说出乎所有人意料，有点不相信，毕竟学长们都那么优秀，但也是对我们一起努力的回报吧。很开心。我们一起去了鸟巢，之前是跟着爸爸一起去的，都很开心。回来的火车上觉得过得好快，早早睡了，也是不舍得一直以来的自己突然闲下来吧。很感谢创新和小黑，我们的组长，还有小黑真的很感谢，两个一年没有回家的人，大一就这么努力我真的发自内心的佩服。可以说不是小黑，我现在挂了好几科了吧。他们都是很优秀的人很幸运，遇到了他们。每个人的方向都不一样却在android实验室相遇，都是孩子，还有对我们很好的学长学姐。为什么都是实验室的内容，我也不知道。

也许花了太多的时间懒惰，后半年自己没学到什么东西，只是没事的时候看看一些技术博客，但是动手实践很少，这里真的想吐槽，什么电路，大物，概率真的不想学，还有实验，我的天。在学长的推荐下自己开始去看《开发艺术探索》，在imooc上买了《android网络编程》没事的时候也是开始学习一写深入点的东西，真心觉得android基础没意思。只有涉及到源码层才是interesting。当然网络也是android学习中一个特别有意思的东西。了解了一点点前端的东西，自己的博客也把架子搭建好，嗯，这个以后放一写好干货吧。嗯，还有在网络学习的过程中避免不了的要搭服务器，每一次尝试都是新领域的打开，java设计模式，rxjava，rxAndroid，自己慢慢学习的也只有这些了，还有好多想学，真的来不及了，虽然自己现在是真的菜，真的菜，真的菜，真的菜，真的菜，真的菜，真的菜，真的菜，我的2016为什么这么的快？在2016快结束的时候在“诺诺”的交流中我发现python真的好神奇啊，于是开始python的“玩耍”，当然知识基础语法现在会一点点，因为自己很心疼钱，所以把一写钱投资在imooc中，是为了提醒自己不能浪费，这些东西都要学习。还有就是自己买了一个手机嗯，原来的r7也不错，准备回家给妈妈。

嗯，寒假来了，在外面的孩子都可以回家了。这次回家自己不想像以前那样一直玩了，大家都长大了，那些发小很想他们，一年我们没有在一块好好玩了吧，不知道会不会聚齐。

!好好陪爸爸妈妈！

虽然理想很美好，但是现实毕竟很残酷。。。。。。。以上至少完成一半（flag：不完成直播吃屎，自己也忘了欠了多少了）

感谢2016遇到的每一个人，同时也感谢那些匆匆教会我东西离去的人，现实教会了我很多，也不再是那个喜欢念旧的人，现在的自己更喜欢未来的自己，爱自己，爱家人。自己还很渺小，有些承诺也只是之后的笑柄，2017我会加倍珍惜自己身边的每个人。

还是有些伤感，一年就这么结束，2017又会有怎样的改变呢？？？？？？？？？？？？？？？？？？？？？？？？？？？
